1.What you'll do

By the end of this page you will have:

  • R + Positron (or RStudio) installed locally.
  • The course's R packages installed: tidyverse, igraph, tidygraph, DBI, RSQLite.
  • Built a 4-node directed network by hand and computed degree and betweenness centrality on it.
  • Re-run the exact same code in the browser-based R Playground as a sanity check.

The goal isn't to teach R or tidygraph thoroughly — that's what the labs are for. It's to make sure your environment works and you've seen the basic shape of a network-analysis pipeline before you start the first real lab.

2.Install R + Positron

R is the language. Positron is the editor we recommend — same Posit folks who built RStudio, multi-language, AI-aware. If you already love RStudio, stay with it. If you live in VS Code, that works too.

The condensed install lives on the Environment Setup page → with screenshots and OS-specific notes.

Locked-down work laptop? Use the playground for the full course. Every lab and the project can be done in-browser.

3.Install course packages

One command. Run it once; it takes 1–3 minutes:

# run once after R is installed install.packages(c("tidyverse", "igraph", "tidygraph", "DBI", "RSQLite"))

If the install fails on igraph, that's almost always missing system libraries on your machine. See the FAQ entry on install failures.

Then load the libraries you need at the top of your session:

library(tidyverse) library(igraph) library(tidygraph)

4.Your first network — typed by hand

Build a tiny 4-node directed supply chain: Factory A ships to B and C; both ship to D.

# nodes: an id and an attribute nodes <- tibble( name = c("A", "B", "C", "D"), tier = c("raw", "sub", "sub", "final") ) # edges: from -> to plus a weight edges <- tibble( from = c("A", "A", "B", "C"), to = c("B", "C", "D", "D"), weight = c( 10, 8, 7, 9) ) # assemble into a tidygraph and print: g <- tbl_graph(nodes = nodes, edges = edges, directed = TRUE) g

You should see something like this:

# A tbl_graph: 4 nodes and 4 edges # A directed simple graph with 1 component # # Node Data: 4 × 2 name tier <chr> <chr> 1 A raw 2 B sub 3 C sub 4 D final # # Edge Data: 4 × 3 from to weight <int> <int> <dbl> 1 1 2 10 2 1 3 8 3 2 4 7 # … with 1 more row

Quick sanity plot — won't be pretty, but confirms the graph is what you think it is:

plot(g, vertex.size = 25, edge.arrow.size = 0.6)
What just happened? A network is its node list and its edge list. Everything else — the picture, the centrality numbers, the analytics — is derived from those two tables. That's the whole conceptual foundation of the course.

5.Your first centrality

tidygraph adds a mutate()-style verb for graph computations. Pipe the graph through, attach degree and betweenness as new node columns, then pull the result back to a tibble:

g |> activate(nodes) |> mutate( degree = centrality_degree(mode = "all"), betweenness = centrality_betweenness() ) |> as_tibble()
# A tibble: 4 × 4 name tier degree betweenness <chr> <chr> <dbl> <dbl> 1 A raw 2 0 2 B sub 2 1 3 C sub 2 1 4 D final 2 0

What this tells you in one sentence each:

  • Degree centrality counts a node's direct connections. All four nodes have degree 2 here — the network is too symmetric for degree to distinguish them.
  • Betweenness centrality counts shortest paths that pass through a node. B and C each sit on the one path from A to D, so they each get a 1; A and D never sit in the middle of anything, so they get 0.

This is the punchline of the Centrality & Criticality lab in miniature: degree and betweenness measure different things, and the "busiest" node by degree is often not the most critical by betweenness.

Pipe note: we use the native |> pipe (R 4.1+). The older magrittr %>% also works if you prefer it, but the course defaults to |>.
