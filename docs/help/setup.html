Get the Tools
Working.

One page that walks you through installing the editor, the language, and the course packages — then verifies the install with a three-line "hello network" snippet. Plan on 20–30 minutes. Skip ahead if you already have a working R or Python setup.

Five steps
  1. 1Pick your language
  2. 2Install Positron
  3. 3Install R or Python + packages
  4. 4Verify with the "hello network" snippet
  5. 5If something went wrong

1.Pick your language

You pick one. Both are first-class in this course — every lab works in either; the playground runs both. The decision is mostly about what you'd actually use at your job.

  • R — slightly stronger graph-analytics ecosystem (igraph, tidygraph), tidyverse pipelines (|>) read close to English.
  • Python — broader general-purpose use, better for ML-heavy follow-up work, more familiar if you do production engineering.

If you genuinely have no preference, default to R for this course. You can change your mind in week 1.

Don't want to install anything? The browser-based Coding Playground runs both R and Python with the course packages pre-loaded. Same workflow, no install. Good fallback if step 3 won't cooperate.

2.Install Positron

Positron is the editor we recommend. It's by the Posit folks (who built RStudio), works for both R and Python, and has good AI integration. One install, one editor, both languages.

Download: https://positron.posit.co/

The download page has installers for macOS, Windows, and Linux. Install, open it once to make sure it launches, then continue to step 3.

Alternatives that also work fine

  • RStudio — if you already have it and love it, stay with it. R-only, but battle-tested. posit.co/download/rstudio-desktop
  • VS Code + the Python extension or R extension — fine if you live in VS Code already. Add Cursor if you want inline AI.
  • Jupyter — works for Python only. Use whatever you're comfortable with.
The course doesn't grade you on your IDE. Pick the one that gets out of your way.

3.Install R or Python + course packages

Install R

Download from CRAN: cran.r-project.org. Pick the package for your OS, run the installer, accept the defaults. R 4.1 or newer.

Install the course packages

Open Positron (or RStudio), then in the R console run:

install.packages(c("tidyverse", "igraph", "tidygraph", "DBI", "RSQLite"))

That's 1–3 minutes. If it fails on igraph, it's almost always a missing system library — see step 5.

Optional but useful

For nicer plots:

install.packages(c("ggraph", "patchwork", "scales"))

Install Python

You need Python 3.10 or newer. Options, in order of preference:

  • uv (recommended) — github.com/astral-sh/uv. Modern, fast Python+package installer. One command: curl -LsSf https://astral.sh/uv/install.sh | sh (macOS / Linux) or the PowerShell equivalent on Windows.
  • python.org installer — straight from python.org/downloads. Click through.
  • conda / miniforge — if you already use it.

Install the course packages

In your terminal (not in Python), run:

pip install networkx pandas matplotlib numpy scipy scikit-learn

Or with uv (faster):

uv pip install networkx pandas matplotlib numpy scipy scikit-learn

That's 1–2 minutes.

4.Verify with the "hello network" snippet

Three lines that prove (a) the language is installed, (b) the graph package is installed, and (c) the plotting library is installed. Paste it into your editor and run. If you see the words "hello, network!" and a tiny plot pops up, you're done with setup.

R version

library(igraph) g <- make_ring(5) plot(g); cat("hello, network! ", vcount(g), "nodes,", ecount(g), "edges\n")
hello, network! 5 nodes, 5 edges

Python version

import networkx as nx, matplotlib.pyplot as plt G = nx.cycle_graph(5); nx.draw(G, with_labels=True); plt.show() print("hello, network!", G.number_of_nodes(), "nodes,", G.number_of_edges(), "edges")
hello, network! 5 nodes, 5 edges

If both lines printed and a window with a 5-node ring graph popped up, your environment is working. Continue to the R quickstart or Python quickstart.

5.If something went wrong

R · igraph won't install

Almost always missing system libraries.

  • macOS: brew install libxml2 glpk then retry.
  • Linux: sudo apt install libxml2-dev libcurl4-openssl-dev libglpk-dev
  • Windows: the precompiled CRAN binary usually just works — if it doesn't, install Rtools and retry.

Python · pip install fails

  • Most common cause: old Python (you need 3.10+). Check with python3 --version.
  • Corporate proxy? Try uv pip install … (uv handles proxies more gracefully).
  • Permissions error? Use pip install --user … instead.

Plot window doesn't appear

  • In a notebook (Jupyter, Quarto): add %matplotlib inline at the top.
  • In a script: make sure you call plt.show() at the end (Python) or that you didn't accidentally save the plot to a file instead of displaying it.

Still stuck

The FAQ has more troubleshooting. If that doesn't help, the in-browser Playground works without any local install — you can use it for the full course if needed. Or book an office-hours slot and we'll debug together.
