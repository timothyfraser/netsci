🛠 Setup & Tools
I can't decide between R and Python — which should I pick?

Pick the one you'd actually use at your job. Both are first-class in this course — every lab works in both, the playground runs both, and the same case studies render in either.

If you genuinely have no preference, default to R: it has slightly stronger graph-analytics packages (igraph, tidygraph) and the tidyverse pipeline syntax (|>) reads almost like English. Python is just as capable and is the better choice if you're heading toward ML/GNN-heavy work.

You don't have to pick on day one — work through the quickstart for both if you want, then commit by end of week 1.

Positron vs RStudio vs VS Code — what should I install?

Default suggestion: Positron. Same Posit folks who make RStudio, but multi-language (works for both R and Python) and AI-aware. Closest to "just works" for this course.

If you already have RStudio installed and love it, stay with it — R-only, but battle-tested. If you live in VS Code already, VS Code + the R or Python extensions is fine; add Cursor if you want inline AI assistance.

The course doesn't grade you on your IDE. Pick the one that gets out of your way.

Package install failed (igraph, tidygraph, networkx) — what do I do?

The most common causes, in order of frequency:

R, igraph — needs system libraries (libxml2, libcurl, sometimes glpk). On macOS: brew install libxml2 glpk. On Linux: sudo apt install libxml2-dev libcurl4-openssl-dev libglpk-dev. Then retry install.packages("igraph").

Python, networkx — should be friction-free with pip install networkx pandas matplotlib. If you're on a corporate laptop with an old Python, install uv first and use uv pip install.

Either — if your install is on a managed work laptop and packages are being blocked by an enterprise proxy, use the in-browser Playground instead. No install needed, packages pre-loaded.

🔬 During a Lab
My numbers don't match the example. Did I do it wrong?

Probably not. Three possible reasons your numbers are off:

(1) Random seed. If the lab involves random sampling, permutation, or Monte Carlo, your result will differ slightly from the example unless you set the same seed (set.seed(42) or random.seed(42)). The magnitude and direction of the result should match; the exact value won't.

(2) Different network handling. Directed vs. undirected, weighted vs. unweighted, self-loops included vs. dropped — small choices like these change centrality numbers. Re-read the lab's setup paragraph.

(3) Actual bug. If the order of magnitude is off (you're getting 0.4 and the lab says 4,000), check your column references — easy to read the wrong column when both are numeric.

Bottom line: match the story, not the digits.

How do I submit a sketch from my phone?

Photograph the finished sketch with your phone's camera, then upload the photo with your Monday submission. JPG or PNG, doesn't have to be high-res — legibility is what matters.

If you want the photo to look slightly nicer: use the "scan document" mode in Notes (iOS) or Google Drive (Android). They flatten the lighting and crop the page automatically. Not required.

🧪 Playground (in-browser R / Python)
WebR / Pyodide is slow on first load — is this normal?

Yes. The first time you open a playground, the browser downloads ~30MB of the WASM-compiled R or Python interpreter, plus the pre-installed packages. On a typical connection that's ~20–40 seconds. Subsequent visits load in a second or two because the browser caches everything.

The status pill at the top of the playground shows you which phase it's in: Loading runtime → Installing packages → Ready ✓.

My dataset is too big for the playground — what now?

Browser tabs cap at ~2–4 GB of memory, which is plenty for any single course dataset but not enough for a million-row CSV at full resolution. Three options:

1. Pre-aggregate. If your network has a million edges but only 500 nodes, aggregate the edges by node-pair (sum / mean / count) before loading. The playground handles ~100k edges comfortably.

2. Sample. Random-node or snowball-sample the network down to a few thousand nodes. The Sampling lab walks through how to do this without destroying the structure you care about.

3. Use a local IDE. Install Positron locally and run your full analysis there. The playground is a quick-experiment tool, not your primary workstation.

My code disappeared!

Your code is auto-saved to your browser's localStorage, which means it survives a refresh but does not survive clearing your browser data. If you cleared cookies/site data, the code is gone with it.

For anything you don't want to lose: copy it into a local file periodically, or paste it into a GitHub gist. The playground is for quick experiments — not your only copy of important work.

📝 Submissions
What goes in a Monday submission?

The short version of the four items every week (full spec on the Assignments page):

1. A short reflection / interpretation in your own words.
2. The Learning Check answer for that week's lab.
3. Your sketch (photo of the sketchpad page).
4. If it's a project week, the relevant project.R or project.py file plus its 2-page-minimum report.

Due Mondays at 9 AM. Extensions available for SYSEN 5940 in-person students by request.

Do I have to do both R and Python tracks?

No. Pick one and stick with it for the course. Each lab is implementable in either language; the choice is yours. If you want to dabble in the other language to compare, you're welcome to — that's what the playground is for.

🎯 Project
I don't have a dataset and it's end of week 1 — what now?

Email Tim. One will be manufactured to fit your field or chosen industry. Don't wait — the longer you wait, the less time you have to actually run analyses on it.

My network only has 60 nodes — is that OK?

The hard minimum is 100 nodes. Networks of 1,000+ nodes are strongly preferred because that's where the difficulty of real-world data actually starts. A 60-node toy network won't surface the analytical challenges the course is built around.

If you have only 60 nodes because the system you care about really is small (a specific team's communication graph, e.g.), email Tim — there's usually a way to expand the boundary (include adjacent teams, add multi-week temporal slices) that preserves your interest while crossing the 100-node line.

Can I use AI to help write my report?

For code and syntax: yes. For the prose of the report: no — that's how we see your thinking.

Full policy with reasoning: AI Use Policy.
