1.What you'll do

By the end of this page you will have:

  • A working Python 3.10+ install with Positron (or VS Code, or your existing setup).
  • The course's Python packages installed: networkx, pandas, matplotlib, numpy, scipy, scikit-learn.
  • Built a 4-node directed network by hand and computed degree and betweenness centrality on it.
  • Re-run the exact same code in the browser-based Python Playground as a sanity check.

The goal isn't to teach Python or networkx thoroughly — that's what the labs are for. It's to make sure your environment works and you've seen the basic shape of a network-analysis pipeline before you start the first real lab.

2.Install Python + Positron

You need Python 3.10 or newer. Positron is the editor we recommend — same Posit folks who built RStudio, multi-language, AI-aware. VS Code + the Python extension is the close runner-up. If you already have a Python workflow you like, keep it.

3.Install course packages

One command, run in your terminal (not in Python):

# in your terminal, after Python is installed pip install networkx pandas matplotlib numpy scipy scikit-learn

If you prefer the modern resolver, uv is faster:

uv pip install networkx pandas matplotlib numpy scipy scikit-learn

Then in your Python script or notebook:

import networkx as nx import pandas as pd import matplotlib.pyplot as plt

4.Your first network — typed by hand

Build a tiny 4-node directed supply chain: Factory A ships to B and C; both ship to D. Same network as the R quickstart, step by step.

# nodes: an id and an attribute nodes = pd.DataFrame({ "name": ["A", "B", "C", "D"], "tier": ["raw", "sub", "sub", "final"], }) # edges: from -> to plus a weight edges = pd.DataFrame({ "from": ["A", "A", "B", "C"], "to": ["B", "C", "D", "D"], "weight": [ 10, 8, 7, 9], }) # assemble into a networkx DiGraph and check: G = nx.from_pandas_edgelist( edges, source="from", target="to", edge_attr="weight", create_using=nx.DiGraph, ) for _, row in nodes.iterrows(): G.nodes[row["name"]]["tier"] = row["tier"] print(G) print("nodes:", G.number_of_nodes(), "edges:", G.number_of_edges())

You should see something like:

DiGraph with 4 nodes and 4 edges nodes: 4 edges: 4

Quick sanity plot — won't be pretty, but confirms the graph is what you think it is:

pos = nx.spring_layout(G, seed=42) nx.draw(G, pos, with_labels=True, node_size=800, node_color="#39FF14", edge_color="#86efac", arrowsize=18) plt.show()
What just happened? A network is its node list and its edge list. Everything else — the picture, the centrality numbers, the analytics — is derived from those two tables. That's the whole conceptual foundation of the course.

5.Your first centrality

networkx exposes centrality as functions that return {node: value} dicts. Stack them onto your nodes DataFrame:

# raw counts (not normalized) for a tiny graph: deg_raw = dict(G.to_undirected().degree()) btw_raw = nx.betweenness_centrality(G, normalized=False) nodes_out = nodes.assign( degree = nodes["name"].map(deg_raw), betweenness = nodes["name"].map(btw_raw), ) print(nodes_out)
name tier degree betweenness 0 A raw 2 0.0 1 B sub 2 1.0 2 C sub 2 1.0 3 D final 2 0.0

What this tells you in one sentence each:

  • Degree centrality counts a node's direct connections. All four nodes have degree 2 here — the network is too symmetric for degree to distinguish them.
  • Betweenness centrality counts shortest paths that pass through a node. B and C each sit on the one path from A to D, so they each get a 1; A and D never sit in the middle of anything, so they get 0.

This is the punchline of the Centrality & Criticality lab in miniature: degree and betweenness measure different things, and the "busiest" node by degree is often not the most critical by betweenness.
