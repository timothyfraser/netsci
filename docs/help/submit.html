SYSEN 5470 · Help · How to Submit

Push It
Up by Monday 9 AM.

Every Monday you submit the same four things from the previous week — a sketch, a case-study learning check, a code learning check, and your project case study. Canvas holds the receipts; GitHub holds the code. Plan on 15 minutes the first time, three minutes once the rhythm clicks.

Five steps
  1. 1What you're submitting
  2. 2Push your code to GitHub
  3. 3Open the Canvas assignment
  4. 4Paste the right thing in the right box
  5. 5Self-grade and submit

1.What you're submitting

Every Monday the Canvas assignment has the same four text boxes, one per item. Here's the shape:

✏️

Sketch

Photo of every sketchpad prompt assigned to this week.

🧪

Case LC

Learning Check answer from every case study lab last week.

⌨️

Code LC

The console output from running last week's example.R or example.py.

📄

Project

Your project script, dataset, and a 2-page report on this week's chosen case study.

Why one assignment with four boxes? So nothing falls through the cracks. Even an empty box is a record — if you skipped sketches one week, the Canvas box is empty rather than ambiguous.

2.Push your code to GitHub

Your project work lives in your netsci-project repo from GitHub Setup. Before you open Canvas, push everything up:

cd netsci-project git status # anything uncommitted? git add code/03-permutation/ # the week's project case study folder git commit -m "Permutation: week 2 project" git push

Your project case study folder for the week should contain three things:

  • project.R or project.py — runs end-to-end on the dataset alongside it.
  • data/ — the network data you used (CSVs, edgelist, anything the script reads).
  • report.pdf — your 2-page write-up.
Network too big for GitHub? Anything over 100 MB doesn't belong in Git. Add the raw file to .gitignore and instead commit a small data/README.md explaining where to download it. The point is reproducibility, not bytes.

3.Open the Canvas assignment

Each week's Canvas assignment is titled "Week N — Monday submission". Open it on a laptop (not phone) because you'll paste links and upload files.

Before you start filling boxes, line up three browser tabs:

  1. The Canvas assignment.
  2. Your netsci-project repo on GitHub — you'll copy the URL of the case-study folder.
  3. The course site's Assignments page — keep the rubric visible so you can self-grade while you submit.

4.Paste the right thing in the right box

Box 1 · Sketch

  • Photograph each sketch on a flat surface, in even light. Phone camera is fine.
  • Upload the JPGs (or a single multi-page PDF). One file per sketch is easiest to grade.
  • Add a one-line caption per sketch: "Sketch 03 — bipartite supply chain, 8 firms × 12 components."

Box 2 · Case-study Learning Checks

Plain text. One line per lab in this exact format:

centrality: B permutation: C joins: A

You don't need to justify your answer. The LC is a quick check that you read the lab, not an essay.

Box 3 · Code Learning Checks

For each case study, run the matching example.R or example.py from the course repo (your pick — you don't need both):

# R track cd netsci/code/04_centrality Rscript example.R # or Python track cd netsci/code/04_centrality python example.py

Each script prints a single answer at the bottom — a number, a node ID, a short string. Paste it in. Add a screenshot of your console for proof:

centrality: node_27 (betweenness 0.184) permutation: p = 0.012 joins: 2,341 weighted edges
Script errored out? First skim the FAQ; most errors are package-install issues from Environment Setup. If you're still stuck, paste the error message into the box — partial credit beats a missing answer.

Box 4 · Project case study

Three things go in:

  1. GitHub link to this week's project folder: https://github.com/USERNAME/netsci-project/tree/main/code/02-permutation/
  2. A one-paragraph "what I did this week" — the question, the network, the result in one sentence. This is the elevator pitch, not the report.
  3. Your report.pdf as an attachment. Same PDF that's in your GitHub repo.

The full report rubric lives on Assignments. Bring it up in a third tab while you self-grade.

5.Self-grade and submit

Before you hit submit, run the self-grade checklist on the Assignments page. It's a short list of yes/no questions — when you click through them, a "Copy paste-able status block" button appears. Click it, then paste the resulting markdown into the top of Box 4.

This isn't graded; it's a forcing function. Most missed-point situations are catches the self-grade would have flagged.

Submit early if you can. Canvas accepts late submissions but they cost points and shift Tim's grading timeline. There's no bonus for going to the wire.

What happens after you submit

  • You'll get feedback on the project report (rubric + a short paragraph) within five days.
  • LC answers don't get individual feedback — they're a binary "you read the lab" check.
  • Sketches don't get scored individually either — they're scored as a set across the term.
  • If something's missing, Canvas will accept a resubmission until Tuesday 9 AM with a one-day-late penalty.
