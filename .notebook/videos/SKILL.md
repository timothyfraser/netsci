# Describe your visual style

Hand-drawn sketchpad aesthetic, like a thoughtful engineer's notebook. 
Off-white or cream paper background with subtle paper texture. All diagrams 
rendered as if drawn by hand with a black or dark navy fine-tip marker — 
slightly imperfect lines, organic curves, visible pen strokes.

The visual centerpiece of nearly every scene should be a NETWORK DRAWING: 
nodes as hand-drawn circles (some filled, some hollow), edges as 
slightly-curved lines connecting them, occasional labels in handwritten-style 
text. Networks should evolve on screen — nodes appearing, edges drawing 
themselves, highlights spreading across the graph — to illustrate concepts 
like centrality, cascades, clustering, or routing.

Accent color: a single warm highlight color (mustard yellow or burnt orange) 
used sparingly to draw attention to a critical node, a bottleneck edge, or 
a key term. Everything else stays monochrome ink-on-paper.

Typography: clean sans-serif for headers (think Inter or similar), with 
occasional handwritten annotations in the margins — arrows, asterisks, 
"important!" notes — as if a student is taking notes while watching.

Avoid: glossy 3D renders, stock photography, corporate-looking infographics, 
gradient backgrounds, animated character avatars, generic AI-generated 
imagery. The whole video should feel like flipping through a brilliant 
engineer's working notebook.

# How would you like the video to be customized?

Audience: Graduate students in a 3-week intensive Cornell course on 
Network Science for Systems Engineering. Mix of coding novices and 
experienced data scientists. Full-time working professionals.

Tone: Direct, warm, a little dry. Like a TA who genuinely wants you to 
get it. No hype, no hedging filler. Concrete examples grounded in engineered 
systems — supply chains, transit networks, power grids, disaster response.

Structure each scene around a question an engineer would ask, then show 
the network drawing that answers it. Example: "Which station, if it failed, 
would break the most trips?" → draw the transit network → highlight the 
critical node in mustard yellow → show the network fragmenting.

Always connect concepts back to the "four professional skills" framing: 
Identify, Measure, Infer, Predict.

End every video with a sketchpad prompt: a question the viewer should 
pause and draw by hand before moving on. Render that final prompt as if 
written on the notebook page itself.

Pacing: Slower than typical explainer videos. Give the viewer time to 
read each network diagram. This is a thinking course, not a scrolling 
course.
