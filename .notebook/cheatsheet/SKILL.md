Create a LANDSCAPE infographic cheatsheet for graduate students in 
SYSEN 5470 (Network Science for Systems Engineering, Cornell). 

PURPOSE
A one-page reference students can print, save, or pin up. Should be 
scannable in 30 seconds and useful as a working reference during labs.

CONTENT STRUCTURE
Determine the cheatsheet TYPE from the topic below:
- CONCEPT cheatsheet (terminology, methods, when-to-use): Organize as 
  definitions + visual examples + "use when / don't use when" guidance.
- CODE cheatsheet (R functions, syntax patterns): Organize as function 
  name + one-line description + minimal example, grouped by task 
  (e.g., "Build a graph", "Measure centrality", "Visualize").
- WORKFLOW cheatsheet (end-to-end process): Organize as numbered steps 
  with decision points, like a flowchart.

LAYOUT (landscape)
- Top banner: Title + one-sentence "what this is for"
- 3-4 column body with clear section headers
- Bottom strip: "Common mistakes" or "Sketchpad prompt" + course link 
  back to the relevant lab or glossary entry
- White space matters — don't cram

VISUAL STYLE
Match the SYSEN 5470 course website aesthetic:
- Clean sans-serif typography (Inter-style)
- Off-white or cream background, not stark white
- Primary ink: dark navy or near-black
- Single accent color: warm mustard yellow or burnt orange, used 
  sparingly to highlight critical terms or key examples
- Where networks appear, draw them sketchpad-style: hand-drawn circles 
  for nodes, slightly curved edges, organic feel
- Emoji icons as section anchors are welcome (the course site uses 
  🕸️ 📊 🔬 🧭 📏 🔮 🚦 🧩) — match icon to topic
- Avoid: glossy gradients, stock imagery, corporate infographic clutter

ALWAYS INCLUDE
- At least one small NETWORK DIAGRAM illustrating the concept, drawn 
  sketchpad-style
- Links back to relevant course material from the sources — cite the 
  specific lab, case study, or glossary entry where this topic lives
- A "Sketchpad prompt" at the bottom: one question the student should 
  draw by hand to test understanding
- The "engineer's lens": one line on why a systems engineer would 
  care about this

TONE
Direct, concrete, no filler. Like a TA's working notes, not a textbook.

GROUNDING
Pull definitions, examples, and links exclusively from the course 
sources in this notebook. If the topic isn't well-covered in the 
sources, say so rather than inventing.
